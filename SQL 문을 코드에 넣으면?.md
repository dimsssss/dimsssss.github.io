회사에서 prisma  ORM을 사용하고 있었는데 배포할 때 prisma에서 에러가 나서 배포 스크립트가 실패를 하게 되었다. 그래서 prisma를 제거하는 작업을 진행하게 되었다.

기존에 ORM 메서드로 데이터베이스에 동작하는 코드를 sql문으로 전부 바꿔야 했다. 여기서 드는 의문은 '코드에 sql문이 들어가면 어떤 장,단점이 발생하는가' 이다. 무조건적으로 '코드에 sql문을 넣으면 안돼'가 아닌 코드에 sql 문장이 있을 때 장단점을 비교하고 저렇게 하는 게 더 좋은 코드다 라고 설명하고 싶었다.

## SQL문은 코드가 아니다
개발자 각자 주력인 프로그래밍 언어로 코드를 작성하지만 SQL로 기능을 구현하는 개발자는 없을 것이다. 전혀 다른 문법의 언어가 섞여 있다는 것이 문제라는 주장을 본적이 있는데 왜 섞여 있으면 안된다는 걸까?
