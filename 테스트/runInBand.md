모노 레포로 구성되어 있는 프로젝트에 여러 개의 서비스가 같이 올라가있다. 예를 들어 하나의 github 프로젝트에 다수의 서비스 코드가 관리되고 있고 각각의 서비스 코드는 개별 개발 서버 설정을 갖고 있다.
A 서비스는 1번 포트, B 서비스는 2번 포트... 이런 식이다.

하나의 프로젝트에 모든 코드를 관리하다보니 테스트 코드 초기화도 한 곳에서 하게 되었다. 우리는 jest, supertest를 이용한 e2e 테스트 코드를 작성하였고 공통으로 사용하는 객체를 초기화 코드에서 생성한 후 global로 등록하였다.

테스트 코드를 실행하였더니 잘되던 테스트 코드가 실패를 하였다. 별도의 디렉터리에서 문제의 테스트 코드를 분리해서 실행을 하니 성공하였다.
